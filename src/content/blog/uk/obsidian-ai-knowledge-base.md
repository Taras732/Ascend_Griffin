> Якщо ти ведеш кілька проектів, працюєш з конфіденційними даними і втомився шукати інформацію по 7 місцях — це коротко про мій підхід.
>
> Базовий setup — один вечір. Звичка — тиждень.

## Чому це важливо

5 років тому "знайти потрібну інформацію" означало години пошуку по Google Drive, Slack-тредах, Notion, Apple Notes і трьох версіях документа з підписом `_final_v3_FINAL`.

Зараз AI може прочитати твою базу знань за секунди і відповісти в твоєму контексті. Але є умова — база має існувати в одному місці і у форматі, який AI може читати безпечно.

Цей пост — короткий огляд того, як я це зробив для себе, і перший конкретний крок. Деталі по кожному блоку — окремі статті, які виходитимуть на цьому ж блозі.

## Філософія в одному реченні

> Дані — твої. Інструмент — змінний. Формат — `.md`, бо він відкриється через 10 років у будь-якому редакторі.

З цього випливає все інше: vault лежить локально, version control через git, AI працює з твоїми файлами на твоїй машині — без витоку у хмарне тренування, без vendor lock-in на роки накопичених записів.

## Стек — 3 інструменти

| Шар | Інструмент | Роль | Ціна |
|---|---|---|---|
| **База знань** | [Obsidian](https://obsidian.md) | Markdown-редактор, граф, wikilinks, плагіни | Free (personal) |
| **AI-агент** | Claude Code у VS Code | Працює з vault'ом як зі звичайним проектом | Pro: $20/міс або API |
| **Версійність** | Git + приватний GitHub | Історія правок, sync між пристроями | Free (private repos) |

Нічого зайвого. Все, що понад це, — додається коли реально треба.

## PARA — як організувати папки

PARA = **P**rojects · **A**reas · **R**esources · **A**rchives. Метод Tiago Forte, тестований мільйонами користувачів. У моєму vault'і виглядає так:

```
00_Inbox/         <- сюди падає все нове, доки не розклав
10_Projects/      <- активні проекти з дедлайнами і метриками
20_Areas/         <- сфери відповідальності без дедлайну (родина, фінанси, здоров'я)
30_Resources/     <- довідкові матеріали, нотатки книг, шаблони
40_Archive/       <- завершене / неактивне (не видаляти — переносити сюди)
99_Meta/          <- шаблони, daily notes, system files
```

Чотири робочі правила: нові файли — у Inbox, раз на тиждень — Inbox to zero, не створювати файлів у корені vault'у, архів — це не видалення.

## Перший конкретний крок — встанови Obsidian (10 хвилин)

1. Завантаж з [obsidian.md](https://obsidian.md).
2. Створи новий vault: назви `MyKnowledgeBase`. Збережи у локальній папці, наприклад `D:/Obsidian/MyVault`.
3. Створи 5 базових папок: `00_Inbox`, `10_Projects`, `20_Areas`, `30_Resources`, `40_Archive`. Плюс папку `99_Meta` з підпапкою `Templates`. І порожній файл `00_Dashboard.md` у корені.
4. Відкрий `Settings → Files & Links → Default location for new notes` → `00_Inbox`. Це найважливіше налаштування з усіх — щоб нічого ніколи не падало в кореневу папку випадково.

Все. У тебе є vault, готовий до наповнення.

## Куди далі

Базовий setup — це лише фундамент. Реальна цінність системи приходить, коли підключиш AI, налаштуєш sync, виробиш ритуал щотижневого огляду і навчишся правильно ставити промпти. Кожне з цих рішень — окрема тема, яку я розпишу глибше у наступних статтях.

Найближчі заплановані матеріали:

- **Підключення Claude Code до vault'у** — VS Code, `CLAUDE.md`, як AI пам'ятає контекст між сесіями.
- **Git + Obsidian без терору** — навіщо це і як налаштувати без бою з командним рядком.
- **Безпека і NDA** — що НЕ кладемо у vault і як працює AI з конфіденційними даними.
- **Sync між пристроями** — порівняння варіантів: офіційний Sync, GitHub, Tailscale + домашній сервер.
- **Плагіни, які я реально використовую** — після року тестів, без культу карго.
- **Templater і шаблони** — як перетворити vault у operating system.
- **Щоденний і тижневий ритуал** — без цього навіть найкраща структура мертва.

## Часті помилки на старті

Дуже коротко, без розгорнутих пояснень:

1. **Перевантажена структура з першого дня.** Почни з PARA + Inbox. Решта виростає природньо.
2. **Файли в корені vault'у.** Не роби. Тільки дашборд і (пізніше) `CLAUDE.md`.
3. **Ігнорування git.** Без історії правок втратиш роботу при першому ж конфлікті sync.
4. **Спокусa "all-in-one" сервісів.** Notion, Roam, Logseq — кожен зав'язаний на свій формат. Markdown — універсальний.
5. **Видаляти стару Inbox-нотатку, бо "це сміття".** Архівуй. Через рік знадобиться.
