> Слово "git" звучить як щось для програмістів. Насправді — це просто спосіб мати приватну копію твоїх файлів в інтернеті, з історією змін. У цій статті жодного рядка коду в терміналі, обіцяю.

## 3 сценарії, які трапляються частіше, ніж здається

Перш ніж говорити "як" — давай чесно про "навіщо". Бекап здається параноєю, поки не станеться одне з трьох:

1. **Ноут зламався.** Найбанальніше. Розлита кава, впав, диск помер, вкрали. Будь-яке з цього — і твої нотатки за пів року роботи зникають разом з пристроєм. iCloud Drive і OneDrive іноді рятують, але часто синхронізували криво, і ти отримуєш половину файлів.
2. **Файл випадково видалений.** Ти або хтось з рідних, хто користується тим самим ноутом. Натиснув не туди, очистив корзину, файл зник. У Obsidian є локальна історія, але вона не вічна.
3. **Vault зламався після оновлення.** Рідко, але буває: оновлення плагіна, конфлікт синхронізації між пристроями, дивна збій-комбінація — і файли є, але читаються пошкоджено.

У всіх трьох випадках рятує одне: **друга копія, яка лежить не на твоєму ноуті**. Не на тому ж диску, не в тому ж домі. В інтернеті, у приватному сховищі, з історією — щоб можна було відкотитись на стан тижневої давності.

## GitHub — приватне сховище в інтернеті

GitHub — це сервіс, де ти можеш мати **приватні** сховища (репозиторії) безкоштовно. "Приватне" означає — тільки ти бачиш вміст, ніхто інший. Це не публікація, це бекап.

Реєстрація:

1. Зайди на [github.com](https://github.com), натисни **Sign up**.
2. Імейл, пароль, ім'я користувача. Все.
3. На безкоштовному плані ти отримуєш необмежену кількість приватних репозиторіїв. Цього достатньо для vault'у на роки вперед.

Створи **приватний** репозиторій (наприклад, з назвою `obsidian-vault`):

1. Натисни **+** в правому верхньому куті → **New repository**.
2. Назва: `obsidian-vault` (або як хочеш).
3. Поставити галочку **Private** — критично, не Public.
4. Не додавай README, .gitignore, ліцензію — створи пустий.
5. **Create repository**.

Запам'ятай URL цього репозиторію — щось типу `https://github.com/[твій-username]/obsidian-vault`. Знадобиться у Кроці 4 нижче.

## Obsidian Git plugin — налаштування через UI

Тепер найкраще: ти не будеш використовувати ні термінал, ні командний рядок. Усе через звичайні налаштування Obsidian.

### Крок 1 — Встановити плагін

1. У Obsidian натисни шестерню зліва внизу — це **Settings**.
2. Зліва обери **Community plugins**.
3. Якщо вперше — натисни **Turn on community plugins** (Obsidian попередить, що ти ставиш сторонній код — погоджуйся).
4. Натисни **Browse**, у пошуку введи "**Obsidian Git**". Це плагін від `denolehov` / `Vinzent03`.
5. **Install** → **Enable**.

### Крок 2 — Встановити Git на ноут

Плагін технічно потребує програми Git на твоєму комп'ютері. Це разова установка, після — забуваєш про неї.

- **Windows:** скачай [git-scm.com/download/win](https://git-scm.com/download/win), запусти інсталятор, усі галочки за замовчуванням — OK. Просто Next-Next-Install.
- **Mac:** відкрий Terminal один раз (Spotlight → Terminal), введи `git --version` і натисни Enter. Mac запропонує встановити **Command Line Tools** — погоджуйся.

Після установки **закрий і відкрий Obsidian** заново — щоб плагін побачив Git.

### Крок 3 — SSH-ключ без бою

SSH-ключ — це наче "ключ від квартири", який дозволяє твоєму ноуту безпечно з'єднуватись з GitHub. Без нього треба було б щоразу вводити пароль.

Найпростіший спосіб згенерувати — попросити Claude:

```
Згенеруй SSH-ключ для з'єднання з GitHub.
1. Створи новий ed25519 ключ з імейлом [твій імейл].
2. Покажи, де він лежить і що мені робити далі —
   куди скопіювати публічний ключ на GitHub.
3. Не виводь приватний ключ у відповідь.
```

AI створить ключ, покаже шлях до файлу, відкриє публічну частину для копіювання. Ти береш цей публічний рядок, копіюєш, йдеш на GitHub → Settings (твоя аватарка) → **SSH and GPG keys** → **New SSH key** → вставляєш → **Add**.

Все. Тепер твій ноут і GitHub знають один одного.

### Крок 4 — Прив'язати vault до репозиторію

1. У Obsidian відкрий Command Palette (`Ctrl+P` / `Cmd+P`).
2. Введи "**Obsidian Git: Clone an existing remote repo**".
3. AI попросить ввести URL. Введи SSH-варіант: `git@github.com:[твій-username]/obsidian-vault.git` (зверни увагу: `git@`, не `https://`).
4. Виконай команду.

Альтернативно — якщо vault вже існує і ти не хочеш починати з нуля, простіше попросити Claude:

```
Мій vault уже містить файли. Прив'яжи його до існуючого
приватного репо на GitHub: [URL]. Зроби перший комміт
з усіма поточними файлами і запуш у репозиторій.
```

AI зробить ініціалізацію, перший комміт і відправку на GitHub. Перевір на сторінці репозиторію в браузері — файли мають з'явитись.

## Auto-commit кожні 10 хвилин

Тепер найприємніше: налаштувати так, щоб тобі взагалі не треба було думати про бекап. Плагін робитиме все сам.

У Obsidian: **Settings → Obsidian Git**. Знайди розділ **Auto** і налаштуй:

- **Vault backup interval (minutes)**: `10` — кожні 10 хвилин комміт і пуш.
- **Auto pull interval (minutes)**: `10` — якщо ти редагуєш з іншого пристрою (про це — у наступних статтях), він підтягне зміни.
- **Pull updates on startup**: `enabled`.
- **Commit message**: можна лишити дефолт або щось своє, типу `vault: {{date}}`.

Все. Тепер кожні 10 хвилин твій vault тихо бекапиться. Без терміналу, без твоєї уваги.

## Як відкотитись на стан тижневої давності

Найкраще у git'і — це **історія**. Кожен комміт — це знімок усього vault'у в певний момент. Якщо випадково видалив важливе три дні тому — можна повернути.

Простіше за все — попросити Claude:

```
Я випадково видалив файл [шлях до файлу] кілька днів тому
і вже зробив комміти після цього. Знайди останню версію
цього файлу в git-історії і відновтись її в поточний стан vault'у.
```

AI пройде історією, знайде, відновить. Якщо хочеш зробити сам — у Obsidian Git є команда **Show diff** у Command Palette, де можна перегортати історію коммітів. Але для першого разу — простіше через AI, він знає синтаксис.

## Часті помилки

**1. Зробив репозиторій публічним замість приватного.** Найгірша помилка — твої нотатки бачить весь інтернет. Перевір зараз: іди на сторінку репо на GitHub. Має бути позначка **Private**. Якщо **Public** — Settings → внизу **Danger Zone** → **Change visibility** → Private.

**2. Поклав у vault великі файли (відео, ISO, дампи).** Git створений для тексту. Якщо запушити файл на 500 МБ — комміти стануть повільними, ліміти GitHub впиратимуться. Великі файли — окремо, у хмарі (Google Drive, Dropbox), не в vault'і.

**3. Конфлікти при роботі з двох пристроїв одночасно.** Якщо ти редагуєш один і той самий файл на ноуті й телефоні без синхронізації між ними — git створить конфлікт. Не страшно, але треба розуміти. Про це — у статті про синхронізацію.

## Що буде далі

Тепер vault бекапиться сам. Можеш видихнути — навіть якщо ноут впаде в річку, твої нотатки за пів року роботи лежать у безпечному приватному сховищі з історією. Але є інше питання, яке варто закрити **до** того, як почнеш заливати туди клієнтські документи: що взагалі **можна** і що **не можна** класти у vault.
