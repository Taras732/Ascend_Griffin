> Якщо ти ведеш кілька проектів, працюєш з конфіденційними даними і втомився шукати інформацію по 7 місцях — цей гайд для тебе.
>
> Базовий setup — один вечір. Звичка — тиждень. Повний потенціал — місяць.

## Чому це важливо саме зараз

5 років тому "знайти потрібну інформацію" означало години пошуку по Google Drive, Slack-тредах, Notion, Apple Notes, поштовій скриньці та трьох версіях документа з підписом `_final_v3_FINAL`.

Зараз AI може прочитати твою базу знань за секунди і відповісти в твоєму контексті. Але є умова — база має існувати в одному місці і у форматі, який AI може читати безпечно.

Цей гайд показує, як я побудував таку систему для себе і використовую її щодня — щоб вести кілька компаній і клієнтських проектів без втрати контексту.

## 1. Філософія: чому локальна база виграє

**Проблема, яка повертається щоразу:**

| Симптом | Реальна причина |
|---|---|
| "Де я це писав?" | Знання розкидані по 5-7 сервісах |
| "Це остання версія?" | Немає одного source of truth |
| "Що ми вирішили місяць тому?" | Контекст помер у Slack-треді |
| "Не можу дати клієнту AI бо там NDA" | Хмарний AI тренується на твоїх даних |
| "Перехід з Notion / Asana = пекло" | Vendor lock-in на роки даних |

**Що вирішує локальна база знань на Markdown:**

- **Один формат** — `.md` відкриється у будь-якому редакторі через 10 років.
- **Один source of truth** — файл у git, історія правок видима.
- **AI працює локально з твоїми даними** — без витоку в тренінг.
- **Гнучкість стека** — змінився інструмент, файли залишились.
- **Швидкість** — `Ctrl+O` → знайшов нотатку за секунди.

> Правило: дані — твої. Інструмент — змінний.

## 2. Стек: 3 інструменти і нічого зайвого

| Шар | Інструмент | Роль | Ціна |
|---|---|---|---|
| **База знань** | [Obsidian](https://obsidian.md) | Markdown-редактор, граф, wikilinks, плагіни | Free (personal) |
| **AI-агент** | Claude Code у VS Code | Працює з vault'ом як зі звичайним проектом | Pro: $20/міс або API |
| **Версійність** | Git + приватний GitHub | Історія правок, sync між пристроями | Free (private repos) |

**Опційно (для просунутих):**
- **Tailscale** — приватна мережа щоб мати vault на телефоні без хмари
- **Domestic server / NAS** — централізований sync вдома
- **Obsidian Git plugin** — auto-commit/pull кожні 10 хв

## 3. Структура PARA — за 30 хвилин

PARA = **P**rojects · **A**reas · **R**esources · **A**rchives. Метод Tiago Forte, тестований мільйонами користувачів.

```
00_Inbox/         <- сюди падає все нове, доки не розклав
10_Projects/      <- активні проекти з дедлайнами і метриками
  ├── Project_A/
  │   ├── 01_Vision/
  │   ├── 02_Discovery/
  │   ├── 03_Roadmap.md
  │   └── Tasks/
  ├── Project_B/
  └── ...
20_Areas/         <- сфери відповідальності без дедлайну
  ├── Health/
  ├── Finance/
  ├── Family/
  └── Career/
30_Resources/     <- довідкові матеріали, нотатки книг, шаблони
40_Archive/       <- завершене / неактивне (не видаляти!)
99_Meta/          <- шаблони, daily notes, system files
00_Dashboard.md   <- головна точка входу
CLAUDE.md         <- інструкція для AI про твій vault
```

**Правила, які заощадять місяці хаосу:**

1. **Нові файли — завжди в `00_Inbox/`**, поки не знаєш точно куди.
2. **Раз на тиждень** — 30 хв на розклад Inbox по папках (Weekly Review).
3. **Не створюй файли в корені** — крім дашборду і CLAUDE.md.
4. **Архів — НЕ видалення.** Закрив проект — переніс у `40_Archive/`. Через рік знадобиться.

## 4. Базовий setup за вечір

**Крок 1: Встанови Obsidian** (10 хв)
- Завантаж з [obsidian.md](https://obsidian.md)
- Створи новий vault: `MyKnowledgeBase`
- Збережи у локальній папці, наприклад `D:/Obsidian/MyVault`

**Крок 2: Створи скелет PARA** (10 хв)
- 5 базових папок: `00_Inbox`, `10_Projects`, `20_Areas`, `30_Resources`, `40_Archive`
- Папка `99_Meta` з підпапкою `Templates`
- Файл `00_Dashboard.md` у корені

**Крок 3: Налаштуй базові плагіни** (15 хв)
`Settings → Community plugins → Browse`:
- **Templater** — шаблони для нових нотаток
- **Dataview** — запити по нотатках як по базі даних
- **Daily Notes** (core) — щоденні нотатки в `99_Meta/Daily_Notes`
- **Obsidian Git** — auto-commit і sync через GitHub

**Крок 4: Створи приватний GitHub repo** (10 хв)
```bash
cd D:/Obsidian/MyVault
git init
git add .
git commit -m "Initial vault"
# Create private repo at github.com/new
git remote add origin git@github.com:USERNAME/my-vault.git
git push -u origin main
```

**Крок 5: Налаштуй Daily Notes** (5 хв)
- `Settings → Daily notes → Folder: 99_Meta/Daily_Notes`
- Format: `YYYY-MM-DD`
- Template: `99_Meta/Templates/Daily.md`

Готово. У тебе є базовий vault.

## 5. Підключення AI: Claude Code у VS Code

**Що це дає:** AI читає твої нотатки, пише чернетки прямо у файли, пам'ятає контекст між сесіями.

**Крок 1:** Встанови VS Code → [code.visualstudio.com](https://code.visualstudio.com)

**Крок 2:** Встанови розширення **Claude Code** (від Anthropic) у VS Code.

**Крок 3:** Залогінься через Anthropic account (Claude.ai Pro $20/міс або API key).

**Крок 4:** Відкрий vault як проект у VS Code:
```
File → Open Folder → D:/Obsidian/MyVault
```

**Крок 5:** Створи `CLAUDE.md` у корені vault'у — це інструкція для AI (див. §7).

Все. Тепер у VS Code є AI, який бачить твою базу знань.

> **Робочий патерн:** Obsidian відкритий для редагування і навігації, VS Code з Claude Code — для роботи з AI. Один і той самий vault, одні й ті самі файли. Зміни одразу видно в обох інструментах.

## 6. Команди, які я використовую щодня

**У Claude Code (slash commands):**

| Команда | Що робить |
|---|---|
| `/init` | Створює CLAUDE.md з аналізом твого vault |
| `/clear` | Очищає контекст для нової задачі |
| `/review` | Рев'ю змін у відкритому файлі/PR |
| `/help` | Список усіх доступних команд |

**Власні промпт-патерни (приклади з реальної роботи):**

```
Прочитай мій vault і знайди всі згадки про проект X.
Зведи в один summary з посиланнями на джерела.
```

```
На основі моїх нотаток за минулий тиждень напиши drafted-звіт
у форматі "Done / In progress / Blocked / Next week".
Збережи у 99_Meta/Daily_Notes/Weekly_Review_YYYY-MM-DD.md
```

```
Подивись мою папку 10_Projects/Client_X/ і скажи де я не закрив петлю —
які issues відкриті, які рішення без власника, які дедлайни горять.
```

```
Перетвори оцей brain dump (вкладено) у структуровану нотатку
з заголовками і wikilinks на існуючі нотатки у vault.
```

**В Obsidian (швидкі дії):**

- `Ctrl+O` — знайти будь-яку нотатку за назвою
- `Ctrl+P` — командна палітра (всі дії)
- `Ctrl+Shift+F` — глобальний пошук по всьому vault'у
- `[[` — створити wikilink на іншу нотатку
- `Ctrl+G` — відкрити graph view (візуалізація зв'язків)

## 7. Як AI запам'ятовує твій контекст: CLAUDE.md і memory

**CLAUDE.md** — файл-інструкція для AI у корені vault'у. AI читає його КОЖНУ сесію.

**Що писати у CLAUDE.md:**

```markdown
# Мій Vault

## Хто я
Тарас, CEO стартапу, BA-консультант. Львів.
Веду 4 компанії + особисте.

## Мова і стиль
- Українська у всіх відповідях
- Коротко, по суті, без води
- Diff/зміни замість всього файлу
- Зміни у 2+ файлах → спочатку план, потім виконання

## Структура (PARA)
... [опис папок]

## Правила
- Нові файли → 00_Inbox
- Не чіпати .obsidian/
- Без видалення без підтвердження
- Wikilinks для всіх зв'язків між нотатками
```

**Memory (auto-memory)** — Claude Code зберігає важливе про тебе між сесіями автоматично:
- Хто ти, які проекти ведеш
- Що вже обговорювали
- Якісь рішення ухвалені раніше
- Твої переваги по тону, форматах, інструментах

Memory лежить у `~/.claude/projects/<vault-name>/memory/MEMORY.md`.

> **Результат:** через місяць використання AI знає твій контекст краще, ніж новий асистент після 3 місяців онбордингу.

## 8. Безпека і конфіденційність — що НЕ давати AI

**Правило #1: Все, що в vault — на твоєму диску. AI читає локально.**

Anthropic Claude API (як і Claude Code) не тренує моделі на даних, які ти передаєш через API. Це задокументовано в Anthropic Privacy Policy. Але є нюанси.

**Окремі категорії варто тримати поза vault або зашифрувати:**

| Категорія | Куди | Чому |
|---|---|---|
| Паролі, API keys, секрети | Password manager (1Password, Bitwarden) | Ніколи не в plain text |
| Скани паспортів, ID | Зашифрований архів окремо | Не для AI-контексту |
| Дані третіх сторін під суворим NDA | Окремий vault, AI вимкнений | Defense in depth |
| Платіжні картки, CVV | Тільки в банку | Ніколи нікуди не пишемо |

**Що захищає базовий setup:**

1. **Vault локально** — не в публічній хмарі.
2. **Приватний GitHub** — repo доступний тільки тобі.
3. **SSH-only access** — без HTTPS passwords.
4. **`.gitignore`** для секретів: `.env`, `*.key`, `secrets/` — ніколи не комітити.

**Додатково (опційно):**
- VeraCrypt-том на диску для super-sensitive нотаток
- Cryptomator для шифрування папки при sync через Google Drive

| Варіант | Складність | Контроль | Multi-device | Ціна |

| **Google Drive / OneDrive sync** | ⭐ | 60% | ✅ | $2-5/міс |
| **Obsidian Sync (офіційний)** | ⭐ | 95% (E2E) | ✅ | $4/міс |

**Моя комбінація:** локальний vault на Windows-ноуті + приватний GitHub (sync через Obsidian Git plugin кожні 10 хв) + домашній сервер з тим самим vault'ом для AI-агентів і доступу з телефона через Tailscale.

## 10. Перехід на Google Drive, якщо локально незручно

Якщо ноут один і ти його носиш — локально + Git вистачає. Якщо хочеш Google Drive — ось як:

**Крок 1:** Створи папку `MyVault` всередині Google Drive desktop sync.

**Крок 2:** При створенні vault'у в Obsidian — обери цю папку.

**Крок 3:** Important — додай у `.gitignore` і Google Drive виключення:
```
.obsidian/workspace.json   # часто змінюється, створює conflicts
.obsidian/cache
.trash/
```

**Крок 4:** Conflicts handling — Google Drive іноді створює `(conflicted copy)` файли. Раз на тиждень очищай їх.

**Крок 5:** AI працює так само — Claude Code відкриває папку як проект, не важливо чи вона у Google Drive.

**Trade-offs Google Drive:**
- Втрачаєш graph view performance на великих vault'ах
- Conflicts при одночасному редагуванні з кількох пристроїв
- Дані шифруються Google, але не end-to-end

**Рекомендація:** локальний + приватний GitHub потужніший і безпечніший. Google Drive — як fallback для тих, хто не хоче розбиратись з Git.

## 11. Щоденний і тижневий ритуал

**Daily (10 хв на день):**
- Ранок: відкрий вчорашню Daily Note, перенеси "Next" у сьогодні
- Протягом дня: все нове — у `00_Inbox/` як `quick_note_topic.md`
- Вечір: 5 хв на розклад Inbox по папках

**Weekly Review (30 хв у п'ятницю/неділю):**
- Очисти `00_Inbox/` повністю
- Огляд кожного активного проекту в `10_Projects/`
- Перенеси завершене у `40_Archive/`
- Створи `Weekly_Review_YYYY-MM-DD.md` з підсумками

**Monthly Review (60 хв на місяць):**
- Аудит структури — папки що розрослись, теми що повторюються
- AI-промпт: "Подивись мій vault і скажи де я витрачаю найбільше часу. Які теми повторюються? Які проекти давно не оновлювались?"
- Архівуй усе, до чого не повертався 90+ днів

## 12. Часті помилки на старті

1. **Перевантажена структура з першого дня.** Почни з PARA + Inbox. Решта виростає природньо.
2. **Копіювання чужих vault-шаблонів.** Спочатку зрозумій свої потреби — потім додавай.
3. **Файли в корені vault'у.** Не роби. Тільки дашборд і CLAUDE.md.
4. **Ігнорування git.** Без історії правок втратиш роботу при першому ж конфлікті sync.
5. **Спокусу від "all-in-one" сервісів.** Notion, Roam, Logseq — кожен зав'язаний на свій формат. Markdown — універсальний.
6. **Не писати CLAUDE.md.** AI без контексту = просто чат. CLAUDE.md перетворює його у твого асистента.
7. **Видаляти стару Inbox-нотатку, бо "це сміття".** Архівуй. Через рік знадобиться.
8. **Боятись AI з конфіденційними даними.** Прочитай privacy policy → працюй через API → дані не йдуть у тренінг.
